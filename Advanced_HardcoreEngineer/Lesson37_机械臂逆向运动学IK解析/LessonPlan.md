# 教师专用教案：Lesson 37 - 机械臂逆向运动学 (IK)

> **⚠️ 内部机密 / 教师专属**
> 本教案为讲师授课指南。如果说 FK 是验证机器人的空间坐标，那么 IK 就是机器人的“大脑指令生成器”。这是整个机械臂课程体系中数学难度最高、最容易出现代码崩溃的一节课。讲师需要利用余弦定理和图解法将算法具象化，并重点防范“域错误(Domain Error)”带来的系统性死机。

## 一、 课程核心目标与背景知识

### 1.1 课程定位
上节课的 FK 让机器臂有了感知能力（知道自己在哪）。但这节课，我们要赋予它**执行能力**。
当我们使用视觉系统（OpenCV）发现了一块位于 $(120, 100, 50)$ 的红色木块时，系统必须瞬间倒推出三个电机应该各自旋转多少度去抓它。这就是**逆向运动学 (Inverse Kinematics, IK)**。跨越了 IK，机械臂才算真正活了过来。

### 1.2 讲师必备背景知识
* **IK 为什么比 FK 难一万倍？**：
  * FK 是唯一的：给你一组固定角度，机械爪只能在一个固定的点。
  * IK 是非唯一的：我想摸桌子上的水杯，我的手肘可以朝上弯曲，也可以朝下弯曲。同一个目标坐标，电机可能有 2 种甚至 8 种不同的角度组合（多解性）。
* **算法流派：解析法 vs 数值法**：
  * **解析法 (Analytical)**：我们本节课用的方法。利用三角形几何关系和余弦定理，一步到位算出精确角度公式。极其快速，非常适合嵌入式系统，但只适用于特定结构的简单机械臂（如 3 或 4 自由度）。
  * **数值法 (Numerical/Jacobian)**：用于复杂的人形骨骼。利用导数迭代逼近，极其消耗 CPU，且可能找不到最优解。
* **奇点 (Singularity) 与 工作空间 (Workspace)**：不是所有的坐标都能抓。如果给的坐标离得太远，手臂拉直了也够不着；或者正好在底座的正上方（此时底座转任意角度都没用），这些都会导致数学计算直接崩溃（比如除以 0，或者对大于 1 的数求反余弦）。

## 二、 专业名词解释 (Glossary)

授课时，请在黑板上画出“大臂+小臂”构成的三角形，并解释以下名词：

* **余弦定理 (Law of Cosines)**：初中数学的巅峰。已知三角形的三条边长度，可以求出任意一个内角的度数。这是解析法 IK 的绝对核心。
* **反三角函数 (Arc Tangent)**：已知坐标求角度的工具。注意，**绝对不要提 `math.atan()`，只教 `math.atan2(y, x)`**。因为 `atan` 分不清四个象限，而 `atan2` 是聪明的，它能自动根据正负号判断是在 $180^\circ$ 还是 $-180^\circ$。
* **肘部朝上/朝下 (Elbow Up / Elbow Down)**：多解性的具象化表达。数学上对应的是求解方程时开平方根取“正号”还是“负号”。
* **空间插补 (Interpolation)**：怎样让机械臂画一条笔直的直线？千万不能同时启动两个电机。必须把一条直线切成 1000 个小点，用 IK 算出这 1000 个点对应的电机角度，然后高频发送。

## 三、 标准授课话术体系 (Talk Tracks)

### 环节一：引出非线性的多解性灾难
> **🗣️ 教师话术建议：**
> “同学们，我现在要求你们把右手食指放在课桌的右上角。好，停住别动。
> 现在看看你们各自的手臂姿势——有的人手肘是抬起的，有的人手肘是贴在桌子上的。这就是 IK 的恐怖之处：**答案不是唯一的！** 如果你是写代码的人，机械臂因为多解性，从上一个点到下一个点时手肘突然反转 $180^\circ$，它可能会当场把旁边的显示器砸个粉碎！”

### 环节二：解析法，把三维降成二维
> **🗣️ 教师话术建议：**
> “三维空间太复杂了，但我们可以取巧。我们把机械臂的动作拆开。底座的电机（Joint 1）只负责左右转，大臂（Joint 2）和小臂（Joint 3）只负责上下动。
> 所以，底座转多少度？只要看 X 和 Y 坐标，一个 `atan2(y,x)` 就搞定了！底座转过去以后，剩下的问题就变成了大臂和小臂组成的平面三角形求角度问题。三维瞬间变成了二维平面几何！”

### 环节三：敬畏工作空间极限
> **🗣️ 教师话术建议：**
> “如果我让你去抓月亮，你能抓到吗？你的手臂没有那么长。在代码里，如果目标距离（斜边）大于了大臂加上小臂的总长度，数学上这就变成了一个无效的三角形。此时如果你强行让电脑算角度，电脑会直接崩溃。所以在写 IK 的第一步，必须进行合法性拦截！”

## 四、 课堂任务与核心代码演示

### 4.1 课前环境与前置科普
在动手写代码前，进行环境准备：
1. **统一软件环境**：打开 **VS Code**（纯 Python 环境），确保同级目录下有上节课写的 `fk_solver.py` 文件（待会用来验证）。
2. **黑板几何推导预演**：在黑板上画出二维的 $(r, z)$ 平面。告诉学生，原坐标 $(X,Y)$ 经过勾股定理合并为了一个水平距离 $r$ ($r = \sqrt{x^2 + y^2}$)，这样就把 3D 彻底降维成了 2D。

### 4.2 核心代码分段拆解与编写
引导学生新建 `ik_solver.py`，并跟随以下步骤逐段写入并理解代码：

```python
import math

# ---------------------------------------------------------
# [第一段] IK 函数骨架与底座角度解算
# ---------------------------------------------------------
def calculate_ik_3dof(x, y, z, L1=80, L2=120, L3=150):
    """
    3自由度机械臂解析法逆向运动学
    输入: 目标坐标 X, Y, Z
    输出: 对应的电机角度 theta1, theta2, theta3
    """
    try:
        # 1. 计算底座旋转角 theta1
        # 直接利用 X 和 Y 计算水平面上的夹角，千万不要用 atan(y/x)，会丢失象限信息
        theta1 = math.atan2(y, x)
        
        # 降维打击：将三维坐标转换为以底座关节为原点的二维柱坐标系 (r, z_eff)
        r = math.sqrt(x**2 + y**2)   # 目标点在水平面的投影距离
        z_eff = z - L1               # 减去底座本身的高度，得到有效的垂直高度

# ---------------------------------------------------------
# [第二段] 防御机制：工作空间拦截检测
# ---------------------------------------------------------
        # 目标点到肩关节的直线距离的平方 (勾股定理)
        hypotenuse_sq = r**2 + z_eff**2
        
        # 【拦截灾难】如果目标距离大于两根手臂伸直的总长度，引发异常
        if hypotenuse_sq > (L2 + L3)**2:
            # 抛出 ValueError，绝不能让程序继续往下算，否则 acos 会崩溃
            raise ValueError("Target Unreachable (Out of Workspace limits!)")

# ---------------------------------------------------------
# [第三段] 余弦定理求解手肘角度 (Theta 3)
# ---------------------------------------------------------
        # 根据余弦定理：c^2 = a^2 + b^2 - 2ab*cos(C)
        # 我们已知 a=L2, b=L3, c^2=hypotenuse_sq，现在反推 cos(C)
        cos_theta3 = (hypotenuse_sq - L2**2 - L3**2) / (2 * L2 * L3)
        
        # 为了防止浮点误差导致 cos_theta3 略微大于 1.0，进行强制夹逼限幅
        cos_theta3 = max(min(cos_theta3, 1.0), -1.0)
        
        # 利用 atan2 计算真实的 theta3 角度 (这里我们选择取负根，代表“手肘朝上”的安全姿态)
        # sin^2 + cos^2 = 1 -> sin = sqrt(1 - cos^2)
        sin_theta3 = -math.sqrt(1 - cos_theta3**2)
        theta3 = math.atan2(sin_theta3, cos_theta3)

# ---------------------------------------------------------
# [第四段] 几何拼图求解肩膀角度 (Theta 2)
# ---------------------------------------------------------
        # 肩膀的总角度等于目标点的绝对仰角(alpha) 减去 内部小三角形的偏角(beta)
        alpha = math.atan2(z_eff, r)
        
        # 内部偏角 beta 也是通过反正切计算得出
        beta = math.atan2(L3 * math.sin(theta3), L2 + L3 * cos_theta3)
        
        theta2 = alpha - beta
        
        # 将弧度(Radians)转换回人类能看懂的角度(Degrees)并返回
        return math.degrees(theta1), math.degrees(theta2), math.degrees(theta3)
        
    except ValueError as e:
        print(f"\n[SYSTEM ALERT] IK Execution Failed: {e}")
        return None, None, None
```

## 五、 突发情况应急预案 (Troubleshooting Protocols)

由于 IK 对物理极限毫无妥协，哪怕差 0.01 毫米，程序都会当场死亡。

### 🚨 灾难 1：Math Domain Error (数学定义域崩溃)
* **学生表现**：程序突然抛出红色的 `ValueError: math domain error`。
* **核心原因**：学生给了超过机械臂极限臂展的 X,Y 坐标。在没有做第二段工作空间拦截机制的情况下，程序强行计算 `math.acos(1.05)`。反余弦函数的值域只能是 `[-1, 1]`，一旦传入 `1.05`，底层 C 语言库就会直接引发致命崩溃。
* **教师应急预案**：
  1. 走下台，检查学生是否漏抄了 `if hypotenuse_sq > (L2 + L3)**2:` 拦截代码。
  2. 检查 `cos_theta3 = max(min(cos_theta3, 1.0), -1.0)` 是否被漏写。因为即使没有超出臂展，极其微小的浮点误差也可能算出 `1.000000000000002`，这同样会引发崩溃。强制夹逼限幅是保命的手段。

### 🚨 灾难 2：原点崩溃导致分母为 0
* **学生表现**：当输入坐标 `X=0, Y=0, Z=200` 时，程序可能报错除以 0，或者底座电机角度返回错乱。
* **核心原因**：当机械臂完全垂直站立在原点正上方时，处于**奇异点 (Singularity)**。此时底座无论转多少度，爪子都在同一个地方。数学上，`atan2(0, 0)` 的结果其实是未定义的（虽然 Python 默认返回 0）。同时，如果内部存在除以 X 或 Y 的代码，就会引发分母为 0 异常。
* **教师应急预案**：
  1. 向学生科普奇异点锁死现象（Gimbal Lock 的兄弟）。在工业操作中，我们绝对会避免让机械臂回到原点正上方。
  2. 在代码开头加入拦截：`if abs(x) < 0.1 and abs(y) < 0.1: x = 0.1`，强行给一个极小的容差。

### 🚨 灾难 3：机械爪砸向了桌子 (多解错乱)
* **学生表现**：算出来的坐标能通过 FK 验证，但是在实际机械臂上跑的时候，手肘是往地下弯曲的，砸坏了结构。
* **核心原因**：在计算 theta3 时，`sin_theta3` 我们选用了负号 (`-math.sqrt()`)。正号和负号对应了“肘部朝上”和“肘部朝下”两种解。如果选错了符号，机械臂就会朝着违背物理常理的方向运行。
* **教师应急预案**：
  1. 修改 `sin_theta3 = math.sqrt(1 - cos_theta3**2)` （把负号去掉或者加上）测试。
  2. 告诉学生，在高级机器人学中，判断选哪个解，需要结合关节的上一次状态，寻找“转动角度最小、能量最省”的那一个解。
