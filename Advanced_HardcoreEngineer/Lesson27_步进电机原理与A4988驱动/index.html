Lesson 27 | 步进电机原理与A4988驱动

教师专属授课系统界面。深入剖析步进电机内部磁场构造与细分驱动原理，利用ESP32与A4988构建工业级非阻塞运动控制框架。拒绝 delay 阻塞，引入硬件定时器中断。

01 专业名词黑板解释 (Glossary)

NEMA 17

工业标准尺寸，17代表法兰边长1.7英寸（42mm）。学生口中的“42步进电机”即指此类。

微步细分 (Microstepping)

A4988的魔法。将原本1.8°的一个物理整步劈成16份，消除低频共振，极度提升精度。

ISR (中断服务)

独立于主循环的神圣特权代码。定时器一到，单片机放下手头所有工作立刻执行的后台幽灵。

02 标准授课话术体系 (Talk Tracks)

🗣️ 环节一：引入痛点 (降维打击新手)

“同学们，回想一下我们之前做的小车。让小车走 10 厘米然后停下，你们是靠 delay() 时间猜的对吧？如果电池没电，它就跑不到。真正的工程师不靠猜！今天引入的步进电机，你发一个脉冲它就转 1.8 度，绝不讨价还价！”

🗣️ 环节二：代码降维打击 (抛弃 delay)

“立刻删掉你们代码里的 delay！在真实的机器视觉系统中，当你 delay 的时候，CPU 睡着了，系统就瞎了。我们要把单片机劈成两半，主循环处理数据，定时器在后台每 500 微秒自动翻转电平，这就是工业级的非阻塞架构！”

03 核心代码演示 (Non-blocking ISR)

在屏幕上展示以下代码，引导学生逐行编写并理解 `volatile` 的物理意义。

main.cpp — PlatformIO Workspace
#include <Arduino.h> // 硬件引脚定义 #define DIR_PIN 19 #define STEP_PIN 18 #define EN_PIN 5 // 【极其重要】中断变量必须加 volatile 防止编译器优化 hw_timer_t * timer = NULL; volatile bool step_state = false; volatile int target_steps = 3200; volatile int current_steps = 0; // 中断服务函数 (ISR) - 加载至内存极速执行 void IRAM_ATTR onTimer() { if (current_steps < target_steps) { step_state = !step_state; digitalWrite(STEP_PIN, step_state); if (!step_state) current_steps++; // 脉冲计数 } } void setup() { pinMode(DIR_PIN, OUTPUT); pinMode(STEP_PIN, OUTPUT); digitalWrite(DIR_PIN, HIGH); // 配置定时器: 1MHz频率 (1微秒Tick一次) timer = timerBegin(0, 80, true); timerAttachInterrupt(timer, &onTimer, true); timerAlarmWrite(timer, 500, true); // 每500微秒翻转一次 timerAlarmEnable(timer); }

04 灾难级应急预案 (Protocols)

灾难 1：A4988 芯片冒烟/炸裂
学生表现： 惊呼，闻到焦糊味，看到火花。
核心原因： 在通电状态下插拔了步进电机线！产生几十伏特反向电动势瞬间击穿。
» 拔掉总电源，更换 A4988。
» 警告全班：“带电插拔，芯片自费！”
灾难 2：ESP32 无限重启 (Panic Error)
学生表现： 串口监视器无限刷出红色寄存器崩溃乱码。
核心原因： 学生在 ISR 中断内部写了 delay() 或 Serial.println()，导致看门狗超时复位。
» 检查并清空 `onTimer` 中的多余代码。
» 强调铁律：“中断函数内只做数学和引脚翻转！”
