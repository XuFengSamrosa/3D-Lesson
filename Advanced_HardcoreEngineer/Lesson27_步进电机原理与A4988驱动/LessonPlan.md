## 一、 课程核心目标与背景知识

### 1.1 课程定位
在之前的课程中，学生们已经掌握了直流电机（DC Motor）的开环 PWM 调速。然而，在自动化分拣线、3D打印机、机械臂等**需要极高空间位置精度**的场景中，直流电机毫无用武之地。本节课是学生迈向“精密运动控制”的第一步，通过引入步进电机（Stepper Motor），让硬件具备精确到毫米甚至微米级的定位能力。

### 1.2 讲师必备背景知识
* **开环与闭环的区别**：步进电机默认是开环控制（盲打），我们给多少个脉冲，它就走多少步。这就要求我们的软件发出脉冲时必须极其稳定（不阻塞），且扭矩必须足够（Vref调参），否则一旦发生“丢步”，系统将彻底失去对位置的掌握。
* **为什么抛弃 `delay()`**：Arduino 新手最喜欢用 `delay()` 来控制电机频率。但 `delay()` 会让单片机进入死循环阻塞。在真正的工业级机器人中，单片机还要同时处理视觉通讯、传感器反馈。一旦阻塞，整个系统就会瘫痪。**必须引入硬件定时器中断 (Timer Interrupt)**。

## 二、 专业名词解释 (Glossary)

授课时，请确保在白板上写下这些名词，并用专业且通俗的语言解释：

* **NEMA 17**：一种工业标准尺寸。17代表其安装法兰的边长为 1.7 英寸（约42mm）。学生常说的“42步进电机”其实就是指 NEMA 17。
* **步距角 (Step Angle)**：电机接收一个脉冲所转动的物理角度。标准两相四线步进电机通常是 $1.8^\circ$。意味着转满一圈（$360^\circ$）需要 $360 / 1.8 = 200$ 个整步脉冲。
* **微步细分 (Microstepping)**：A4988 驱动器特有的魔法。通过控制两个线圈中电流的比例（形成正弦波），把原本一个物理整步（$1.8^\circ$）再强行劈成 16 份。开启 16 细分后，转一圈就需要 $200 \times 16 = 3200$ 个脉冲。它能极大地降低电机的运行噪音和低频共振。
* **ISR (Interrupt Service Routine)**：中断服务函数。它是独立于 `loop()` 主循环之外的代码。当定时器时间一到，单片机会像被“打断”一样，放下手头所有工作，立刻去执行 ISR，执行完再回来。
* **梯形加减速 (Trapezoidal Profile)**：解决物理惯性的算法。电机静止时惯性极大，直接给高频脉冲会让它憋住不转（发丝丝声）。必须经历：缓慢起步加速 $\rightarrow$ 高速匀速运行 $\rightarrow$ 缓慢减速刹车。

## 三、 标准授课话术体系 (Talk Tracks)

### 环节一：引入痛点（激发兴趣）
> **🗣️ 教师话术建议：**
> “同学们，回想一下我们之前做的小车。当我们想让小车往前走 10 厘米然后停下，我们是怎么做的？（停顿等学生回答：用 delay 时间来估算）。没错，靠猜。如果电池满电，它跑得远；没电了，它跑得近。这种误差如果在机械臂上，可能你的机械臂早就把桌子砸穿了。
> 真正的工程师是不靠猜的。今天，我们要引入工业界的绝对主力——步进电机。我们发一个脉冲，它就转 $1.8^\circ$，绝不讨价还价，绝不多走一丝一毫！”

### 环节二：剖析 A4988 与细分魔法
> **🗣️ 教师话术建议：**
> “步进电机里面有两个粗壮的线圈，需要的电流非常大，ESP32 脆弱的引脚直接连上去会瞬间烧毁。我们需要一个保镖，这就是 A4988 驱动板。
> 注意它上面的 MS1、MS2、MS3 三根针脚。如果我们把它们全部接高电平（HIGH），我们就开启了‘微步细分’魔法。把原来跨一大步的动作，切分成了 16 个极其细腻的太空步。这不仅提高了精度，还能让那个烦人的电机啸叫声消失。这叫用算法解决物理缺陷！”

### 环节三：代码降维打击（抛弃 delay）
> **🗣️ 教师话术建议：**
> “我看有很多同学还在用 `delayMicroseconds()` 写代码。请立刻删掉它！在真实的机电系统中，这叫‘自杀式代码’。当你 delay 的时候，你的机器视觉检测到障碍物了吗？没有，因为你的 CPU 睡着了！
> 接下来我们要学习的东西叫硬件定时器（Hardware Timer）。我们要把单片机劈成两半，主循环该干嘛干嘛，而定时器在后台每隔 500 微秒，像幽灵一样自动去翻转电平。这就叫**非阻塞式并行架构**！”

## 四、 课堂任务与核心代码演示

### 4.1 课前环境与前置科普
在动手写代码前，请按照以下流程引导学生进行环境准备：
1. **统一软件环境**：指导全体学生打开 **VS Code** 并进入 **PlatformIO IDE** 插件（或 Arduino IDE）。确保开发板型号选择为 `DOIT ESP32 DEVKIT V1`，波特率设置为 `115200`。
2. **硬件接线自检**：在大屏幕上打出接线图。要求同桌之间互相检查：ESP32 的 19 引脚接 A4988 的 DIR，18 接 STEP，5 接 EN。**再次大声强调：接线期间必须拔掉外部总电源！**
3. **科普定时器概念**：在黑板上画一个时钟，告诉学生 ESP32 内部有 4 个以 80MHz 狂飙的“定时炸弹”。我们设置分频器为 80，让它变成 1MHz（即 1 微秒 tick 一次）。当它数到我们设置的报警值（例如 500）时，就会瞬间触发警报（产生中断），去执行翻转电平的任务。

### 4.2 核心代码分段拆解与编写
引导学生新建 `main.cpp`，并跟随以下步骤逐段写入并理解代码：

```cpp
#include <Arduino.h>

// ---------------------------------------------------------
// [第一段] 硬件引脚映射与全局变量定义
// ---------------------------------------------------------
#define DIR_PIN 19   // 方向控制引脚
#define STEP_PIN 18  // 脉冲控制引脚 (给一个脉冲走一步)
#define EN_PIN 5     // 芯片使能引脚 (低电平激活 A4988)

// 定义定时器指针
hw_timer_t * timer = NULL; 

// [极其重要] 在中断中使用的变量必须加 volatile 关键字！
// 这可以防止编译器过度优化，确保每次都从内存中读取最新值。
volatile bool step_state = false;    // 记录引脚当前是高还是低
volatile int target_steps = 0;       // 我们要电机走的总步数
volatile int current_steps = 0;      // 电机当前已经走过的步数

// ---------------------------------------------------------
// [第二段] ISR 中断服务函数 (硬件级别的特权函数)
// ---------------------------------------------------------
// IRAM_ATTR: 强制将此函数加载到内存(RAM)而非闪存(Flash)中，保证极速执行
void IRAM_ATTR onTimer() {
  // 如果还没走到目标步数，就继续翻转电平
  if (current_steps < target_steps) {
    step_state = !step_state;            // 状态取反 (HIGH变LOW，LOW变HIGH)
    digitalWrite(STEP_PIN, step_state);  // 将状态写入引脚
    
    // 一个完整的脉冲包含一次HIGH和一次LOW，我们在变回LOW时计为走了一步
    if (!step_state) {
      current_steps++;
    }
  }
}

// ---------------------------------------------------------
// [第三段] 系统初始化配置
// ---------------------------------------------------------
void setup() {
  Serial.begin(115200); // 开启串口通信用于接收电脑指令
  
  // 初始化引脚模式
  pinMode(DIR_PIN, OUTPUT);
  pinMode(STEP_PIN, OUTPUT);
  pinMode(EN_PIN, OUTPUT);
  
  digitalWrite(EN_PIN, LOW);   // 拉低使能引脚，唤醒 A4988 工作
  digitalWrite(DIR_PIN, HIGH); // 默认设置正转方向

  // [定时器核心配置]
  // 参数1: 定时器编号(0-3)；参数2: 分频系数(80MHz/80=1MHz，即1微秒)；参数3: 向上计数
  timer = timerBegin(0, 80, true);
  
  // 将我们写的 onTimer 函数挂载到定时器中断上
  timerAttachInterrupt(timer, &onTimer, true);
  
  // 设置报警值: 500微秒。当定时器数到500时触发中断，且开启自动重装载(true)
  timerAlarmWrite(timer, 500, true);
  
  // 启动定时器，电机随时准备接收目标指令
  timerAlarmEnable(timer);
  
  Serial.println("System Ready. Waiting for target steps...");
  
  // 先给它设定走 3200 步 (如果是16细分，刚好转1圈)
  target_steps = 3200;
}

// ---------------------------------------------------------
// [第四段] 主循环 (完全非阻塞)
// ---------------------------------------------------------
void loop() {
  // 主循环不需要任何 delay() 或脉冲翻转代码！它被完全解放了！
  // 我们可以在这里轻松地处理串口数据，而不会影响电机运转。
  
  if (Serial.available() > 0) {
    String input = Serial.readStringUntil('\n');
    int new_target = input.toInt();
    if (new_target > 0) {
      current_steps = 0;          // 重置当前步数
      target_steps = new_target;  // 设置新的目标步数
      Serial.printf("New target set: %d steps\n", target_steps);
    }
  }
}
```

### 4.3 编译、运行与现象观测
1. **代码编译与烧录**：指导学生点击 VS Code 底部的 `→` (Upload) 按钮，将代码烧录至 ESP32。
2. **打开串口监视器**：点击插头图标打开 Serial Monitor。确保波特率匹配为 `115200`。
3. **预期运行结果**：
   * **初始表现**：开机后，串口打印 `System Ready...`，电机自动顺滑地旋转一圈（3200步）后平稳停止。
   * **交互控制**：学生在串口发送框中输入数字（例如 `6400`）并按下回车。串口会立即回复 `New target set: 6400 steps`，电机瞬间启动，匀速旋转刚好两圈后再次精准停止。
   * **核心启发**：引导学生注意，在电机旋转期间，如果连续发送数据，串口响应没有任何卡顿！这向学生们有力地证明了：我们的电机控制正在后台(中断)自动运行，彻底脱离了 `delay` 的泥沼，实现了完美的多任务并行。

## 五、 突发情况应急预案 (Troubleshooting Protocols)

由于涉及到大电流和精密的硬件，本节课是极易发生硬件损坏和程序崩溃的高危区。教师必须随时待命。

### 🚨 灾难 1：A4988 芯片冒烟/炸裂
* **学生表现**：惊呼，闻到焦糊味，甚至看到火花。
* **核心原因**：学生在**主电源未切断的情况下，插拔了步进电机的接线**！这种操作会导致电机线圈产生巨大的反向感应电动势（几十伏特），瞬间击穿 A4988。
* **教师应急预案**：
  1. 立即拔掉总电源。
  2. 换上新的 A4988 模块。
  3. **大声警告全班**：“这是血的教训！任何人插拔电机线之前，必须确认断电。谁再带电插拔，芯片费用自理！”

### 🚨 灾难 2：电机发出极其尖锐的高频啸叫，且轴不转（堵转）
* **学生表现**：电机疯狂尖叫，摸起来发烫，但就是不转。
* **核心原因**：
  1. **频率过高**：初始脉冲给得太快（比如定时器周期设了 100 微秒），没有加减速，电机惯性跟不上。
  2. **电流不足**：A4988 的 Vref 电位器调得太低，输出扭矩不够。
  3. **线序接错**：步进电机四根线中，同相（A+A-）接错了。
* **教师应急预案**：
  1. 让学生拔掉 EN 引脚停止脉冲。
  2. 掏出万用表打到蜂鸣档，测试电机四根线，两两连通的为同一相，必须将它们排在相邻的位置插入驱动板。
  3. 用十字螺丝刀微调 A4988 上的十字电位器，顺时针旋转 $15^\circ$ 增加电流。
  4. 检查代码中的 `timerAlarmWrite` 值是否过小，要求学生改到 1000 以上测试。

### 🚨 灾难 3：ESP32 不断自动重启，串口疯狂打印 Panic Core Error
* **学生表现**：程序完全跑飞，IDE 串口监视器中无限刷出红色的寄存器崩溃乱码。
* **核心原因**：学生在中断服务函数（ISR `onTimer`）内部写了极其耗时的操作，比如 `delay()` 或者 `Serial.println()`。中断函数具有最高系统权限，一旦超时，看门狗（Watchdog）会直接复位重启单片机以保全系统。
* **教师应急预案**：
  1. 检查学生的 `onTimer()` 函数。
  2. 强调铁律：“ISR 里必须速战速决！只能做数学运算和翻转引脚！要打印数据，用全局变量扔到 `loop()` 里面去打印！”
  3. 检查全局变量前是否加了 `volatile` 关键字，防止被编译器错误优化。
