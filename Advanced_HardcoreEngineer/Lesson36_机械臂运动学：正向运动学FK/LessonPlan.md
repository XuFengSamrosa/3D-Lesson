# 教师专用教案：Lesson 36 - 机械臂运动学：正向运动学 (FK)

> **⚠️ 内部机密 / 教师专属**
> 本教案为讲师授课指南。正向运动学（FK）是机器人算法的骨架，涉及大量空间几何与线性代数。讲师必须在课前熟练推导 $4 \times 4$ 齐次变换矩阵的数学意义，并在课堂上把枯燥的矩阵运算具象化为“机械臂的空间接力赛”，带领学生完成从“写代码”到“造数字孪生”的跨越。

---

## 一、 课程核心目标与背景知识

### 1.1 课程定位
如果说之前的电机控制是在造“肌肉”，那从这节课开始，我们要给机械臂注入“灵魂”。机器人无法直接听懂“去抓桌上的水杯（X,Y,Z坐标）”，它只听得懂底座电机转 $45^\circ$、肩膀转 $30^\circ$。
**正向运动学 (Forward Kinematics, FK)** 就是一座桥梁：已知所有关节的旋转角度，推算出机械爪在现实物理空间中的三维坐标。这是所有机械臂算法的绝对起点。

### 1.2 讲师必备背景知识
* **两大坐标空间的鸿沟**：
  * 关节空间 (Joint Space)：电机的角度集合 $[\theta_1, \theta_2, \theta_3]$。
  * 笛卡尔空间 (Cartesian Space)：真实世界的物理坐标 $[X, Y, Z, Roll, Pitch, Yaw]$。
* **齐次变换矩阵 (Homogeneous Transformation Matrix)**：如果只用普通的坐标偏移，很难同时处理“平移”和“旋转”。引入一个 $4 \times 4$ 的矩阵，左上角 $3 \times 3$ 负责旋转姿态，右边 $3 \times 1$ 负责位置偏移。这样，复杂的空间运动就变成了最简单的矩阵乘法。
* **D-H 参数 (Denavit-Hartenberg)**：工业界描述机械臂的标准格式。通过 $a$ (连杆长度)、$\alpha$ (连杆扭转角)、$d$ (连杆偏置)、$\theta$ (关节角) 四个极其精简的参数，就能描述宇宙中任意两个相邻关节的空间关系。

## 二、 专业名词解释 (Glossary)

授课时，请在黑板或白板上画出机械臂简图，并结合以下比喻进行解释：

* **坐标系传递 (Frame Transformation)**：想象接力跑。基座把接力棒（坐标）交给肩膀，肩膀交给手肘，最后交给手抓。每一次交接棒，都要乘以一个变换矩阵。
* **D-H 参数四大金刚**：
  * **$a$ (连杆长度)**：可以理解为骨头的真实长度。
  * **$\theta$ (关节角)**：这就是我们要用代码去控制的电机，唯一会变动的参数。
  * **$d$ 和 $\alpha$**：由于机械臂为了避让，关节可能不在同一个平面上，这两个参数负责描述“骨头的偏移和扭曲程度”。
* **末端执行器 (End Effector)**：其实就是机械臂最前端夹木块的那个“爪子”或者焊枪。我们算的一切坐标，都是为了计算它在哪里。
* **Numpy 库**：Python 科学计算的核武器。不要让学生自己写 `for` 循环去算矩阵乘法，直接调用 C 语言底层的 Numpy 库。

## 三、 标准授课话术体系 (Talk Tracks)

### 环节一：引入痛点（机器人的沟通障碍）
> **🗣️ 教师话术建议：**
> “同学们，你们觉得对于一台机械臂来说，‘伸手拿一个苹果’简单吗？极其困难。
> 因为苹果在桌子上的坐标是 (X=150, Y=200, Z=50)，但是你对机械臂的底座电机喊‘你去 (150,200,50)’，它听得懂吗？听不懂，它是个瞎子，它只知道‘我该转 30 度还是 60 度’。
> 怎么把电机的角度翻译成现实的三维坐标？今天我们要揭晓现代机器人学最核心的密码——正向运动学！”

### 环节二：降维解析齐次变换矩阵
> **🗣️ 教师话术建议：**
> “在三维空间里，物体有两种运动：平移和旋转。如果我们用加减法算平移，用三角函数算旋转，公式会越写越长，最后连电脑都会算崩溃。
> 伟大的数学家发明了 $4 \times 4$ 齐次变换矩阵。它就像一个魔盒，把旋转和平移打包在了一起。只要把基座的矩阵 乘以 肩膀的矩阵 再乘以 手肘的矩阵，瞬间！结果矩阵的右上角，就是机械爪的绝对 XYZ 坐标。这就是线性代数的暴力美学！”

### 环节三：揭秘数字孪生 (Digital Twin)
> **🗣️ 教师话术建议：**
> “学会了这个 FK 算法，我们不仅能控制真实的机械臂。想一想，如果我们在电脑里也建一个同样的矩阵模型，只要实体电机的角度一变，电脑里那个虚拟机械爪的 XYZ 坐标就跟着变。这就是现在马斯克和波士顿动力都在搞的最高端技术——数字孪生！”

## 四、 课堂任务与核心代码演示

### 4.1 课前环境与前置科普
在动手写代码前，进行环境准备：
1. **统一软件环境**：指导全体学生打开 **VS Code**，创建一个纯 Python 的工作区（不需要连接 ESP32 开发板，本节是纯上位机算法课）。
2. **安装核心科学库**：打开 VS Code 的集成终端 (Terminal)，让所有人输入 `pip install numpy`。**注意：必须确保每位电脑都提示 Successfully installed！**
3. **科普弧度制**：在黑板上画一个圆，特别警告：“在人类的世界里，我们说转 $90^\circ$；但在 Numpy 的世界里，它只认弧度。$90^\circ = \pi / 2$。待会在代码里，绝对不能把 90 直接塞进 `math.sin()` 里面，否则你的机械臂会直接穿模！”

### 4.2 核心代码分段拆解与编写
引导学生新建 `fk_solver.py`，并跟随以下步骤逐段写入并理解代码：

```python
import numpy as np
import math

# ---------------------------------------------------------
# [第一段] 核心算法：D-H 矩阵生成器
# ---------------------------------------------------------
# 这是一个通用“模具”。只要输入某两个关节之间的4个物理参数，
# 它就能瞬间吐出一个 4x4 的齐次变换矩阵
def dh_matrix(a, alpha, d, theta):
    # 【灾难预防】第一步永远是把人类的角度(Degrees)转换为机器的弧度(Radians)
    alpha_rad = math.radians(alpha)
    theta_rad = math.radians(theta)
    
    # 严格按照标准 D-H 矩阵公式构建 4x4 Numpy 数组
    # 前 3x3 区域负责旋转，第 4 列负责平移(X,Y,Z)，最后一行永远是 [0,0,0,1]
    matrix = np.array([
        [math.cos(theta_rad), -math.sin(theta_rad)*math.cos(alpha_rad),  math.sin(theta_rad)*math.sin(alpha_rad), a*math.cos(theta_rad)],
        [math.sin(theta_rad),  math.cos(theta_rad)*math.cos(alpha_rad), -math.cos(theta_rad)*math.sin(alpha_rad), a*math.sin(theta_rad)],
        [0,                    math.sin(alpha_rad),                      math.cos(alpha_rad),                     d],
        [0,                    0,                                        0,                                       1]
    ])
    
    # 过滤掉极小的浮点误差 (比如 1.22e-16 变成 0.0)
    return np.round(matrix, decimals=4)

# ---------------------------------------------------------
# [第二段] 针对我们 Cyber-Sorter 机械臂的运动链级联
# ---------------------------------------------------------
def calculate_fk(theta1, theta2, theta3):
    # 根据我们机械臂真实的物理尺寸填入 D-H 参数表
    # a: 连杆长度(毫米), alpha: 连杆扭转角(度), d: 偏置距离(毫米), theta: 当前电机旋转角度(传入的变量)
    
    # 矩阵1: 从大地铁基座 到 关节1 (底座负责水平旋转)
    T1 = dh_matrix(a=0,   alpha=90, d=80, theta=theta1)
    
    # 矩阵2: 从关节1 到 关节2 (大臂负责上下俯仰)
    T2 = dh_matrix(a=120, alpha=0,  d=0,  theta=theta2)
    
    # 矩阵3: 从关节2 到 关节3 (小臂连着机械爪)
    T3 = dh_matrix(a=150, alpha=0,  d=0,  theta=theta3)
    
    # 【高能预警】矩阵级联点乘！
    # 注意：在 Numpy 中，矩阵相乘必须用 '@' 符号，绝对不能用 '*'
    T_end = T1 @ T2 @ T3
    
    # 最终的 T_end 就是末端执行器相对于基座的总变换矩阵
    # 提取第4列(索引为3)的前3行(索引为0,1,2)，这就是最终的现实坐标 [X, Y, Z]
    x = T_end[0, 3]
    y = T_end[1, 3]
    z = T_end[2, 3]
    
    return x, y, z

# ---------------------------------------------------------
# [第三段] 测试与验证
# ---------------------------------------------------------
if __name__ == "__main__":
    # 模拟场景：底座转了45度，大臂抬起30度，小臂下压-30度
    target_t1 = 45
    target_t2 = 30
    target_t3 = -30
    
    # 召唤 FK 算法进行解算
    x, y, z = calculate_fk(target_t1, target_t2, target_t3)
    
    print("\n[CYBER-SORTER DIGITAL TWIN STARTED]")
    print(f"Current Joints : [Base={target_t1}°, Arm1={target_t2}°, Arm2={target_t3}°]")
    print(f"End Effector 3D: X = {x:.2f} mm, Y = {y:.2f} mm, Z = {z:.2f} mm\n")
```

### 4.3 编译、运行与现象观测
1. **代码执行**：指导学生在 VS Code 终端中输入 `python fk_solver.py` 运行程序。
2. **预期运行结果**：
   * 控制台应清晰打印出输入的关节角度，以及经过矩阵连乘算出的最终物理坐标 $X, Y, Z$。
3. **极值验证法（启发式教学）**：
   * 让学生把代码里传入的角度改成 `(0, 0, 0)` 运行一次。
   * **提问**：如果三个电机都在 0 度（机械臂完全水平伸直），算出坐标应该是多少？
   * **验证**：根据连杆尺寸，$X$ 轴距离应当是 $120 + 150 = 270$ 毫米，高度 $Z$ 轴应当是基座高度 $80$ 毫米。如果算出的坐标是 $X=270, Z=80$，说明算法完全正确！

## 五、 突发情况应急预案 (Troubleshooting Protocols)

由于涉及到线性代数和浮点运算，学生极易犯语法和数学常识错误。

### 🚨 灾难 1：使用 `*` 导致矩阵维度崩溃
* **学生表现**：运行代码时，终端大面积标红，抛出类似 `ValueError: operands could not be broadcast together with shapes (4,4) (4,4)` 的致命错误，或者算出的结果是一个乱码矩阵而不是标量坐标。
* **核心原因**：学生在编写矩阵连乘时，由于 Python 惯性思维，使用了 `T_end = T1 * T2 * T3`。在 Numpy 中，`*` 代表“逐元素相乘（Element-wise）”，这就要求两个矩阵结构必须一模一样且违背了线性代数法则。
* **教师应急预案**：
  1. 走下讲台检查代码，将 `*` 修改为 `@`（Python 3.5 以后引入的矩阵点乘专属运算符）。
  2. 如果学生的 Python 版本过低不支持 `@`，指导其修改为古老写法：`T_end = np.dot(np.dot(T1, T2), T3)`。

### 🚨 灾难 2：算出的坐标极度离谱，甚至是负数乱跑
* **学生表现**：给了一个 `(90, 0, 0)` 的直角，结果算出来的 X, Y, Z 全是一大堆无规律的几十的数字，完全不符合物理定律。
* **核心原因**：漏写了 `math.radians()`。学生直接把 `90` 塞进了 `math.cos(90)`。注意，这里的 `90` 被计算机理解成了 **90个弧度（相当于绕地球转了十几圈）**，而不是 $90^\circ$。
* **教师应急预案**：
  1. 立即指向上方 `dh_matrix` 函数的开头。
  2. 严厉检查是否遗漏了 `alpha_rad = math.radians(alpha)` 和 `theta_rad = math.radians(theta)`。加上后结果瞬间恢复正常。

### 🚨 灾难 3：出现了像 `1.2246467991473532e-16` 这种诡异数字
* **学生表现**：惊恐地举手，说自己的 X 坐标算出来不是 0，而是一个带有 `e-16` 的超长小数，怀疑算法算错了。
* **核心原因**：这是计算机科学中的“浮点精度幽灵”。由于计算机使用二进制近似表达 $\pi$（`math.pi`），所以在计算 $\sin(180^\circ)$ 或 $\cos(90^\circ)$ 时，永远无法得到绝对干净的 $0.0$。
* **教师应急预案**：
  1. 恭喜学生发现了计算机底层的秘密，借机科普 IEEE 754 浮点数标准。
  2. 指导学生在 `dh_matrix` 返回时，包上一层 `np.round(matrix, decimals=4)`，把小数点后第 4 位以后的幽灵数字强行抹零。世界再次清静。
