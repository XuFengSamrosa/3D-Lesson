## 一、 课程核心目标与背景知识

### 1.1 课程定位
在前面的课程中，学生已经掌握了电机的开环运动（“闭着眼睛往前走”）。但在真实的工业环境中，机械臂和智能小车会遭遇摩擦力、电池电压骤降、外力碰撞等无数干扰。本节课是机器人的“觉醒时刻”——引入闭环反馈（Closed-loop Feedback）和 PID 算法，让系统长出“眼睛”，具备自我纠错和抗干扰能力。

### 1.2 讲师必备背景知识
* **开环 vs 闭环**：开环是“执行命令不论结果”，闭环是“时刻检查结果并修正误差”。
* **P（比例）的作用与局限**：推力与误差成正比。但当误差很小（比如离目标只有1毫米）时，P 的推力也极小，小到无法克服电机的物理静摩擦力，从而永远停在目标点旁边（稳态误差）。
* **I（积分）的作用与局限**：为了解决 P 的稳态误差，I 会把历史上的微小误差随时间累加。哪怕误差只有 0.1，累加 100 次后推力也会变得巨大，从而彻底消除稳态误差。但如果累加过头，会导致系统冲过头（超调 Overshoot）。
* **D（微分）的作用与局限**：预测未来。根据误差缩小的“速度”（斜率）来产生反向阻尼。如果发现冲向目标的速度太快，D 会提前踩刹车。但 D 对高频噪声极度敏感，传感器哪怕稍微抖动一下，D 都会将其放大为剧烈的电机抽搐。

## 二、 专业名词解释 (Glossary)

授课时，请在黑板上写下这些控制论名词，并配以生动的比喻：

* **Setpoint (目标值) & Measured Value (当前值)**：Setpoint 就是我们想让小车保持的速度（比如 100），Measured Value 是编码器传回来的真实速度（比如 80）。
* **Error (误差)**：永远等于 `Setpoint - Measured Value`。PID 算法所有的计算都基于这个误差值。
* **稳态误差 (Steady-State Error)**：系统稳定后，始终与目标值差了那么一点点。就像你想把水温调到 40 度，结果水温永远卡在 38 度上不去。
* **积分饱和 (Integral Windup)**：一个工业界必须防范的灾难。假如轮子被石头卡死了，无论电机怎么转，误差始终存在，I（积分）就会无限累加到一个天文数字。当石头被搬走时，电机会像疯了一样以最大功率冲刺，根本停不下来。
* **超调 (Overshoot) 与 震荡 (Oscillation)**：超调就是冲过了目标线又倒退回来；震荡就是在目标线前后疯狂反复横跳。

## 三、 标准授课话术体系 (Talk Tracks)

### 环节一：引入痛点（生活场景代入）
> **🗣️ 教师话术建议：**
> “同学们，假设你现在正在高速上开车，限速 120km/h。这就是你的 **Setpoint（目标值）**。如果你只踩下三分之一的油门，车速到了 100km/h 就上不去了，这就是开环控制的失败。
> 为了达到 120，你会怎么做？你会看着时速表（这叫**反馈**），发现还差 20，于是你继续深踩油门。当你发现车速到了 115，快接近目标了，你就会慢慢松开一点油门，防止超速。这就叫**闭环控制**。而把这个老司机的脑部活动翻译成代码，就是伟大的 PID 算法！”

### 环节二：降维解析 P、I、D（弹簧与阻尼器）
> **🗣️ 教师话术建议：**
> “不要被微积分这三个字吓倒。在代码里，它们非常简单。
> **P 就是一根弹簧**，你把它拉得越远（误差大），它拉你回来的力气就越大。
> **I 是一个记仇的小本本**，你差了多少，它就一秒一秒地给你记下来算总账，直到你一点误差都不剩为止。
> **D 就是一个刹车片**，它不管你离目标有多远，它只看你跑得‘有多快’。你跑得越快，它摩擦力越大，专门用来防止你冲过头。”

### 环节三：工程现实的毒打（为什么课本公式不能直接用）
> **🗣️ 教师话术建议：**
> “很多大学生在课本上学了 PID 的连续公式，然后直接写进单片机，结果电机当场崩溃。为什么？因为单片机是离散的（一行行执行代码），而数学公式是连续的。更可怕的是‘积分饱和’。如果小车被按在地上摩擦，你的积分项会累加到爆表。所以真正的工业代码，必须加上‘限幅’和‘抗积分饱和’的护甲！”

## 四、 课堂任务与核心代码演示

### 4.1 课前环境与前置科普
在动手写代码前，进行环境准备：
1. **统一软件环境**：打开 **VS Code + PlatformIO**。本节课我们将使用串口绘图器（Serial Plotter）来直观地看曲线，所以必须确保波特率统一设定为 `115200`。
2. **离散化科普**：在黑板上解释微积分如何变成代码：
   * 积分 (Integral) $\rightarrow$ 累加：`error_sum = error_sum + error`
   * 微分 (Derivative) $\rightarrow$ 差分：`derivative = (error - last_error)`

### 4.2 核心代码分段拆解与编写
引导学生新建 `main.cpp`，逐段编写一个标准的 C++ PID 类：

```cpp
#include <Arduino.h>

// ---------------------------------------------------------
// [第一段] 工业级 PID 类的声明
// ---------------------------------------------------------
class PIDController {
private:
    float Kp, Ki, Kd;               // P, I, D 三个神秘的魔法参数
    float error_sum, last_error;    // 记忆体：历史误差累计(给I用)，上次误差(给D用)
    float out_max, out_min;         // 硬件输出极限 (例如 PWM 最高只有 255)
    
public:
    // 构造函数：初始化参数
    PIDController(float p, float i, float d, float max, float min) {
        Kp = p; Ki = i; Kd = d;
        out_max = max; out_min = min;
        error_sum = 0; last_error = 0;
    }

// ---------------------------------------------------------
// [第二段] 核心计算逻辑 (离散化微积分)
// ---------------------------------------------------------
    // dt: 距离上次执行过了多少时间 (Delta Time)
    float compute(float setpoint, float measured_value, float dt) {
        // 1. 求出当前误差
        float error = setpoint - measured_value;
        
        // 2. 比例项 (P) - 活在当下
        float P_out = Kp * error;
        
        // 3. 积分项 (I) - 记忆过去
        error_sum += error * dt;     // 误差乘以时间就是面积（积分）
        float I_out = Ki * error_sum;
        
        // 4. 微分项 (D) - 预测未来
        float derivative = (error - last_error) / dt; // 误差的变化率（斜率）
        float D_out = Kd * derivative;
        
        // 5. 计算总输出推力
        float output = P_out + I_out + D_out;
        
// ---------------------------------------------------------
// [第三段] 防御机制：限幅与抗积分饱和 (Anti-windup)
// ---------------------------------------------------------
        // 如果计算出的推力大到超出了硬件极限（比如PWM超过255）
        if (output > out_max) {
            output = out_max;
            error_sum -= error * dt; // 【核心防护】拔除本次无效的积分累加
        } else if (output < out_min) {
            output = out_min;
            error_sum -= error * dt; // 【核心防护】拔除反向无效累加
        }
        
        // 保存当前误差，留给下一次给 D 用
        last_error = error;
        
        return output;
    }
};

// ---------------------------------------------------------
// [第四段] 实例化与主循环模拟测试
// ---------------------------------------------------------
// 创建一个PID对象: Kp=2.0, Ki=0.5, Kd=0.1, 输出限制为 -255 到 255
PIDController myPID(2.0, 0.5, 0.1, 255.0, -255.0);

float current_speed = 0.0; // 模拟当前速度
unsigned long last_time = 0;

void setup() {
  Serial.begin(115200);
  last_time = millis();
}

void loop() {
  unsigned long now = millis();
  float dt = (now - last_time) / 1000.0; // 转换为秒
  last_time = now;
  
  float target_speed = 100.0; // 目标速度
  
  // 1. 调用 PID 算法得出修正力度
  float power = myPID.compute(target_speed, current_speed, dt);
  
  // 2. 模拟物理世界：动力让速度增加，同时有摩擦力让速度衰减
  current_speed += (power * 0.1) - (current_speed * 0.05); 
  
  // 3. 按照 Arduino 串口绘图器的格式打印，用空格或逗号隔开
  Serial.print("Target:"); Serial.print(target_speed);
  Serial.print(" Current:"); Serial.print(current_speed);
  Serial.print(" Power:"); Serial.println(power);
  
  delay(50); // 采样周期 50ms
}
```

### 4.3 编译、运行与现象观测
1. **代码编译与烧录**：将代码上传至 ESP32。
2. **打开 Serial Plotter (串口绘图器)**：这不是普通的串口监视器，而是 VS Code 或 Arduino IDE 自带的曲线绘制工具。
3. **预期运行结果**：
   * **观察追踪曲线**：在绘图器中，学生会看到 `Current` 曲线像一条贪吃蛇一样，迅速爬升并紧紧咬住 `Target`（值 100）的直线。
   * **调整参数实验**：让学生修改代码中的 `Kp = 10.0` 重新烧录，他们会直观地看到曲线发生极具破坏性的**高频震荡**；然后调大 `Kd`，又会发现震荡被神奇地抹平了。
   * **核心启发**：通过纯数学的手段改变一条物理曲线的走向，这就是控制论的暴力美学。

## 五、 突发情况应急预案 (Troubleshooting Protocols)

PID 调参在工业界被称为“炼丹”，学生极易陷入参数混乱的深渊，甚至引发硬件剧烈抖动损坏。

### 🚨 灾难 1：系统产生高频剧烈震荡 (疯狂抽搐)
* **学生表现**：电机疯狂正反转抽搐，或者串口曲线呈密集的锯齿状，根本停不下来。
* **核心原因**：**P 增益 (Kp) 设定过大**，导致系统极其敏感，稍微有一点误差就用尽全力去推，结果永远在越过目标线的两端疯狂试探；或者 **D 增益 (Kd) 设定过大** 且传感器有噪声。
* **教师应急预案**：
  1. 立即停止电机或断电，防止机械结构被高频震动震碎。
  2. 指导学生执行**“清零重置法”**：将 Ki 和 Kd 全部设为 0。
  3. 让 Kp 从 0 开始慢慢往上加，直到系统刚开始出现轻微震荡时，将当前的 Kp 数值直接**减半**，作为最终的 P 参数。

### 🚨 灾难 2：响应迟缓，永远达不到目标值 (稳态误差)
* **学生表现**：目标是 100，但曲线爬到 85 就变平了，像是动力枯竭了，永远差那么一点。
* **核心原因**：系统遭遇了固定阻力（比如摩擦力或重力），而比例项 P 产生的推力恰好与阻力抵消了。缺乏积分项 I 来打破平衡。
* **教师应急预案**：
  1. 检查代码中的 Ki 参数是否为 0，或者过小（如 0.0001）。
  2. 缓慢增加 Ki（比如每次加 0.1）。让学生观察曲线，会发现那条停滞的曲线开始缓慢但坚定地向上蠕动，最终完美贴合目标线 100。

### 🚨 灾难 3：抗积分饱和代码写错，导致灾难级延迟
* **学生表现**：用手按死电机的轮子几秒钟，然后松手，电机像疯了一样最高速空转很久很久才慢慢降速。
* **核心原因**：学生在抄写代码时，漏掉了 `error_sum -= error * dt;` 这一行核心的抗饱和（Anti-windup）代码。导致轮子被按死期间，I 项疯狂累计出了天文数字。
* **教师应急预案**：
  1. 结合生活场景解释：“这就像你按住了一个人让他走不动，他心里就在积攒怒气。你按得越久，他怒气值越高。等你松手，他就要发疯暴走。”
  2. 让学生检查第三段防御机制的代码，确保当输出超过 `out_max` 时，立刻停止 `error_sum` 的继续累加。重新烧录后，验证按死再松手，电机能在 1 秒内恢复平静。
